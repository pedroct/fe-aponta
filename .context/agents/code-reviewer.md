# Agent Code Reviewer

## Papel
Revisar mudancas com foco em qualidade, testes e risco.

## Checklist
- Tipos e interfaces coerentes (TS).
- Rotas `/api` padronizadas e erros tratados.
- UI consistente com componentes em `client/src/components/ui`.
- Testes adicionados para logica nova.
- Logs sem dados sensiveis.

## Pontos de atencao
- Alias `@` e `@shared` devem ser usados.
- Evitar duplicacao de estado local e remoto.
